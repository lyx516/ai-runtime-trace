# 二分查找函数 — 规格说明书 (Specification)

## 1. 概述

实现一个 Python 函数 `binary_search`，在**已排序的升序列表**中查找目标值，返回其索引。若目标值不存在，返回 `-1`。

## 2. 功能需求

### 2.1 核心功能

| ID | 需求描述 |
|----|----------|
| FR-01 | 函数接受一个已排序（升序）的整数列表 `arr` 和一个整数目标值 `target`，返回目标值在列表中的索引（0-based）。 |
| FR-02 | 若目标值在列表中出现多次，返回任意一个匹配的索引（不做稳定性要求）。 |
| FR-03 | 若目标值不存在于列表中，返回 `-1`。 |

### 2.2 函数签名

```python
def binary_search(arr: list[int], target: int) -> int:
    """在已排序的升序列表中查找目标值。

    Args:
        arr: 已按升序排序的整数列表。
        target: 要查找的目标整数值。

    Returns:
        目标值在列表中的索引（0-based），若不存在则返回 -1。
    """
```

## 3. 边界条件

| ID | 条件 | 预期行为 |
|----|------|----------|
| BC-01 | 空列表 `[]` | 返回 `-1` |
| BC-02 | 单元素列表，目标值匹配 | 返回 `0` |
| BC-03 | 单元素列表，目标值不匹配 | 返回 `-1` |
| BC-04 | 目标值在列表开头（索引 0） | 返回 `0` |
| BC-05 | 目标值在列表末尾（索引 len-1） | 返回 `len(arr)-1` |
| BC-06 | 目标值小于所有元素 | 返回 `-1` |
| BC-07 | 目标值大于所有元素 | 返回 `-1` |
| BC-08 | 列表包含重复值，目标值存在 | 返回任意一个匹配索引 |
| BC-09 | 列表元素为负数，目标值为负数 | 正常查找并返回索引 |
| BC-10 | 列表长度为 1（最小有效长度） | 正常查找 |

## 4. 输入输出定义

### 4.1 输入

| 参数 | 类型 | 约束 |
|------|------|------|
| `arr` | `list[int]` | 已按升序排序；可为空；元素为任意整数（含负数） |
| `target` | `int` | 任意整数 |

### 4.2 输出

| 类型 | 说明 |
|------|------|
| `int` | 目标值的索引（0-based），不存在返回 `-1` |

### 4.3 示例

| 输入 arr | 输入 target | 输出 | 说明 |
|----------|-------------|------|------|
| `[1, 3, 5, 7, 9]` | `5` | `2` | 中间元素匹配 |
| `[1, 3, 5, 7, 9]` | `1` | `0` | 首元素匹配 |
| `[1, 3, 5, 7, 9]` | `9` | `4` | 尾元素匹配 |
| `[1, 3, 5, 7, 9]` | `4` | `-1` | 不存在 |
| `[]` | `1` | `-1` | 空列表 |
| `[2, 2, 2]` | `2` | `0` 或 `1` 或 `2` | 重复值，任意索引均可 |
| `[-5, -3, 0, 2]` | `-3` | `1` | 负数查找 |
| `[10]` | `10` | `0` | 单元素匹配 |
| `[10]` | `5` | `-1` | 单元素不匹配 |

## 5. 算法要求

- 必须使用**二分查找算法**，时间复杂度 O(log n)。
- 禁止使用 Python 内置的 `bisect` 模块或 `list.index()` 方法。
- 空间复杂度 O(1)（迭代实现，不使用递归以避免栈溢出风险）。

## 6. 错误处理

| ID | 场景 | 处理方式 |
|----|------|----------|
| EH-01 | `arr` 为 `None` | 抛出 `TypeError("arr must be a list")` |
| EH-02 | `arr` 不是 `list` 类型 | 抛出 `TypeError("arr must be a list")` |
| EH-03 | `target` 不是 `int` 类型 | 抛出 `TypeError("target must be an int")` |
| EH-04 | `arr` 未排序（非升序） | 不强制校验（调用方负责传入已排序列表）；若需校验可扩展但不在本 spec 范围内 |
