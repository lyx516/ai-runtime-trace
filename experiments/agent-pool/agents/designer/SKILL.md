
# SKILL: 文件写入规范与错误处理

## 背景
根据反馈，write_file 工具曾因文件路径权限或内容格式问题失败1次。为提高可靠性，遵循以下规范：

### 写入前检查
- 始终使用正确的文件后缀（如 `.md`, `.py`, `.txt`）
- 确保文件路径存在权限（若目标目录不存在，先创建）
- 写入内容必须是**字符串**，避免二进制数据

### 错误重试机制
- 若 `write_file` 返回失败，最多重试 **2 次**，每次间隔 **1 秒**
- 重试前检查文件是否被锁定（如被其他进程占用），可尝试写入临时文件名
- 若连续失败，记录日志并考虑使用 `file_read` 确认文件状态

### 示例
```python
write_file("output/report.md", "# Draft content")
```
避免：`write_file("report", b"binary data")`
